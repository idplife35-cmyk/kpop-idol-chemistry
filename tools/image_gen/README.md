# tools/image_gen/

> Playwright-driven ChatGPT image generation. Reads a queue of JSON prompts and
> saves the produced PNGs to canonical asset paths in the repo.
>
> Fits user policy: external image APIs may be paid → we drive ChatGPT through
> Playwright, same pattern as `analytics/ga_scraper.mjs`.

## Setup (one-time)

```bash
cd ~/project/kpop-idol-chemistry/tools/image_gen
npm install
npx playwright install chromium
```

## Run a single prompt

```bash
node image_gen.mjs \
  --prompt "K-Pop concert poster, neon, no text" \
  --out ../../public/og/blog/foo.png \
  --size 1200x630
```

## Run the queue

```bash
node image_gen.mjs --queue queue
```

The first run pops a Chromium window — sign into ChatGPT once. The session is
saved under `.profile/` and reused. Completed queue files move to `queue/done/`.

## Queue file format

```json
{
  "id": "kebab-case-id",
  "out": "public/og/foo/bar.png",
  "size": "1200x630",
  "prompt": "…",
  "style": "optional style note",
  "priority": 1,
  "notes": "Why this image exists — for human reviewers."
}
```

`out` is resolved relative to the repo root.

## Selectors drift

ChatGPT's DOM changes often. If `image_gen.mjs` times out:

1. Run with the browser visible (default already).
2. Inspect the composer / send / generated-image selectors.
3. Patch `SELECTORS` at the top of `image_gen.mjs`.

## Tribe-intel workflow

When tribe-intel proposes a new image (blog hero, share card variant, group
artwork, ad creative):

1. Add a `queue/NN-{id}.json` file with the prompt + canonical out path.
2. Run `npm run gen:queue` either ad-hoc or as part of weekly publish.
3. Reference the generated image in code (`og:image`, `<img src>`, etc.).
4. Commit both the queue file (audit trail) and the produced PNG.
