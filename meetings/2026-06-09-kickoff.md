## 의제 1: 우리의 1차 부족은 누구인가?

### 각 에이전트 입장 (요약)

**tribe-intel — 글로벌 영어권 K-Pop 팬 (특히 1세대·4세대 메가 팬덤) 우선**
- K-Pop 글로벌화는 2018년 BTS·BLACKPINK 진입 이후 영어권이 압도적 성장. 2026년 현재 K-Pop 트윗의 70%+가 영어. 미국·동남아·라틴아메리카 팬덤이 신규 진입 주도.
- 부족 규범("우리는 OO를 사랑한다는 정체성"): ARMY, BLINK, MOA, ONCE, STAY, NCTzen이 모두 다국적 부족이며, 영어가 부족 공용어다.
- 결정적 비대칭: K-Pop "이름 만들기" 행위 = 정체성 도구. 영어권 팬은 한국식 이름을 "받는 것" 자체가 부족 진입 의식(rite of passage). 한국 팬에게는 진입 의식이 아니다 (이미 한국 이름이 있음).
- 한 줄 입장: **"영어권 팬에게 한국 이름은 부족 정체성 토큰. 한국 팬에게는 그저 재미."**

**product — 글로벌 영어권 우선 (강하게)**
- 리마커블 테스트: 영어권 팬은 한국식 이름·케미 결과를 받으면 트위터/TikTok에 거의 무조건 공유한다. 한국 팬은 공유 동기가 약하다 (정체성 토큰이 아니므로).
- VS Mode 가치: 영어권 팬 간 "내가 BLACKPINK 누구랑 케미가 더 높냐"는 강력한 부족 내 게임. 한국 팬은 비교적 무관심.
- 현재 사이트(영/한 다국어, 65+ 그룹)는 글로벌 영어권을 위해 이미 설계된 것에 가깝다.

**ladder — 글로벌 우선, 단 PWA·푸시 친화도는 일본·한국 우위**
- 결과 공유 → 친구 유입은 영어권에서 압도적 작동. 트위터/TikTok 공유 문화.
- 반면 PWA 설치율은 일본·한국이 더 높음 (모바일 비중). 영어권은 데스크톱·iOS Safari가 많아 PWA가 제한적.
- 권장: 영어권 1차 부족, 단 허락 사다리 본체는 "공유 → 즐겨찾기 → 컴백 알림 이메일" 중심. 푸시는 일본·한국에서만 보조.

**seo — 글로벌 영어권 키워드 시장이 압도적**
- "kpop name generator" / "BTS name generator" / "BLACKPINK name generator" 영어 키워드 월 검색량이 한국어 대비 약 20~50배.
- 경쟁: 영어권에는 경쟁 사이트 다수 (kpopnamegenerator.com 외 3~5개). 그러나 우리가 65+ 그룹 + 게임화 + 케미 점수까지 갖춘 곳은 1~2개.
- 한국어 키워드 "케이팝 이름 생성기"는 시장 자체가 작음 (월 1,000~3,000 검색 추정).
- 인프라: 영어 페이지 안정적 운영, sitemap 정상, AdSense 통합 가능 상태로 보임 (`/public/ads.txt` 존재 확인).
- 한 줄 입장: **"한국 시장은 100만원/월 달성에 PV가 절대 부족하다. 영어권이 유일한 경로다."**

**adops — 글로벌(특히 미국+일본+한국) 트래픽 믹스가 RPM에 결정적**
- AdSense RPM 통상: 미국 $4~10 > 호주/캐나다/영국 $3~5 > 일본 $2~3 > 한국 $1~2 > 동남아 $0.3~1.
- 100만원/월 = $750. RPM $3 (영어권 평균 가정) × 250,000 PV/월 또는 RPM $5 × 150,000 PV/월.
- 한국 시장 단독: RPM $1.5 가정 시 500,000 PV/월 필요 → 사실상 불가능.
- 영어권 우선 + 미국 비중 30%+ 확보 시 PV 150,000~200,000으로 100만원 달성 가능.
- 동남아 트래픽(필리핀·인도네시아) 비중 너무 높으면 RPM 폭락 위험. → 미국·서유럽·일본·한국 균형 필요.

**analytics — 베이스라인 데이터: 이미 글로벌 분포**
- 6/9 GA 홈 스냅샷: 실시간 사용자 싱가포르 1, 한국 1. 7일 활성 269명 중 95% 신규.
- 동종 앱 중앙값 대비 활성 사용자 수가 한참 아래 → 성장 여지 큼.
- 결정적 약점: 주요 이벤트 0 → 전환 추적 자체가 없음. 어디서 사용자가 무엇을 하는지 측정 불가.
- 한 줄 입장: **"지금은 안개 속이다. 부족 결정 전에 GA 이벤트 정의가 P0."**

### 크로스 비판

**6인 모두 "글로벌 영어권" 우세였기에, 형식적 동의가 아닌 "잘못될 수 있는 경로" 점검:**

- **반박 가능성 1 — "영어권 경쟁자가 이미 많다. 후발주자가 100만원 달성?"**
  → 반박: 경쟁자 중 게임화·VS Mode·65+ 그룹 모두 갖춘 곳은 1~2개. 차별화 자산 있음. 그러나 SEO만으로 100만원 PV는 3개월 안 어렵다 → 바이럴(TikTok·트위터) 채널이 보조 필수.
- **반박 가능성 2 — "한국 시장 무시는 K-Pop 본거지 진정성 손실"**
  → 반박: 무시가 아니라 1차/2차 우선순위. 한국어 페이지·다국어 인프라는 유지. 단, 마케팅·콘텐츠 리소스의 80%는 영어권에 투입.
- **반박 가능성 3 — "다중 부족(영·한·일 동시) 가능?"**
  → 반박: Seth Godin 금기 항목 1번 ("타깃을 '모든 K-Pop 팬'으로 정의"). **거부.** 단, 일본어는 Month 3 회고 시 영어권 트래픽 안정화 조건으로 추가 검토.
- **반박 가능성 4 — "1세대(BTS·BLACKPINK) 메가 팬덤은 너무 포화"**
  → 부분 수용: 1세대는 트래픽 절대량 최대, 그러나 신규 4·5세대(NewJeans, IVE, RIIZE, PLAVE, aespa) 팬덤이 입소문 가속도 더 큼. → tribe-intel이 1세대 + 신규 4·5세대를 30:70으로 가중치 부여.

### 팀장 결정

> **결정 1: 1차 부족 = 글로벌 영어권 K-Pop 팬 (특히 4·5세대 메가 팬덤)**
> **결정 2: 한국어 페이지는 보조 채널로 유지. 마케팅·신규 기능 리소스의 80%는 영어권에.**

**근거 (Seth Godin 체크리스트 통과):**
1. **부족에 봉사하는가?** Yes. 영어권 팬에게 한국식 이름은 부족 정체성 토큰.
2. **리마커블한가?** Yes. 트위터/TikTok 공유 동기 강함.
3. **퍼널이 작동하는가?** Yes. 공유 → 친구 유입 → 즐겨찾기 → 푸시·이메일 옵트인 단계 완결.
4. **딥인가?** Yes. 65+ 그룹 + VS Mode + 게임화 = 사용자가 깊이 들어갈 수 있는 구조.

**문 열어둘 것 (피벗 트리거):**
- Month 2 종료 시 일본 트래픽이 10%/월 초과하면 일본어 페이지 추가 검토.
- Month 3 종료 시 한국 트래픽이 30% 초과하면 한국어 마케팅 비중 재조정.
- 그 전까지는 영어권 80% / 한국어 20% 리소스 배분.

## 의제 2: 3개월 월 100만원 — 현실성 검증

### 수식 분해 (analytics + adops 합동 보고)

```
월 100만원 ≈ $750/월
$750 = RPM × 월간 PV / 1000
RPM $3 가정 → 월간 PV 250,000 필요
RPM $5 가정 → 월간 PV 150,000 필요 (영어권 비중 높을 때)
RPM $2 가정 → 월간 PV 375,000 필요 (한국 비중 높을 때)
```

### 베이스라인 → 목표 격차

| 항목 | 베이스라인 (6/9) | 3개월 목표 (9/9) | 배수 |
|------|------------------|--------------------|------|
| 7일 활성 사용자 | 269 | 6,000 | 22× |
| 추정 월간 PV | ~2,000 | 250,000 | 125× |
| 추정 월 수익 | ~$5 | ~$750 | 150× |

### 비판적 검토

**adops의 솔직한 평가:**
- "3개월 안에 100만원은 도전적 목표다. SEO 정상 성장은 6~12개월. 100만원/월은 보통 1년 이상 누적의 결과다."
- "달성 가능 시나리오:
   1. 바이럴 1~2회 성공 (TikTok 트렌드 편승, 신규 그룹 컴백 타이밍 활용)
   2. 또는 Mediavine 등 premium 광고 네트워크 진입 (50,000 PV/월 조건) → RPM 3~5배 점프
   3. 또는 두 가지 동시 달성"

**Seth Godin 입장:**
- "100만원 자체가 본질 목표가 아니다. **부족 깊이가 본질이고 100만원은 결과지표다.** 부족이 깊어졌는데 100만원에 못 미친다면 그건 RPM의 문제. 부족이 못 깊어졌다면 그건 우리 문제."
- "단, 사용자가 명확히 3개월 100만원이라는 부족 외 목표를 부여했다. 우리는 그 목표를 진지하게 추구하되, 부족을 떠나게 하는 방식(가짜 트래픽, 인터스티셜 광고)으로는 추구하지 않는다."

### 팀장 결정

> **결정 3: 100만원/월 목표는 유지. 단, 부족 건강도가 우선 게이트.**
> - 트래픽 가속은 부족이 친구를 데려오는 방식(바이럴)으로만 추구.
> - SEO + 바이럴 + 광고 네트워크 업그레이드의 3축 전략.
> - Month 2 종료 시 100만원 경로에서 25% 이상 이탈 시 일정 연장 또는 스코프 재조정 (자기수정 트리거).

## 의제 3: Week 1 실행 계획 (2026-06-09 ~ 06-15)

### P0 (반드시 6/15까지)

1. **GA 주요 이벤트 정의 + 측정 시작** (analytics + product)
   - 이벤트: `name_generated`, `vs_started`, `result_shared`, `favorite_added`, `pwa_installed`
   - GA4 측정 ID 확인 → 코드에 이벤트 발화 추가 → 주요 이벤트로 GA에서 마킹
   - 6/15 기준 베이스라인 스냅샷 `analytics/baseline-2026-06-15.md` 저장

2. **결과 공유 카드 리마커블 게이트** (product + 팀장)
   - 현재 공유 카드(이미지/링크) 디자인을 팀장이 직접 검토
   - 기준: "이 카드를 받은 친구가 사이트로 올 이유가 있는가? 친구의 친구가 또 올 이유가 있는가?"
   - 통과 시 유지. 미통과 시 product가 1주 안에 재디자인.

3. **재방문 트리거 1개 라이브** (ladder)
   - 후보 A: 즐겨찾기 UX 강화 + "다시 케미 보러 오기" 카피
   - 후보 B: PWA 설치 유도 (안드로이드 모바일 사용자 대상)
   - A/B 1주 비교 → 채택

4. **부족 세계관 v1** (tribe-intel)
   - 4·5세대 메가 팬덤(NewJeans, IVE, aespa, RIIZE, PLAVE) 부족별 내면 서사 1페이지
   - "이 부족이 우리 사이트에서 무엇을 얻고, 어디로 공유하는가" 매핑
   - product + seo에 브리프 전달

5. **AdSense 단가 베이스라인** (adops)
   - AdSense 대시보드에서 현재 RPM, CTR, Fill rate, 광고 단가 7일 평균 추출
   - 광고 위치별 단가 분석 → 가장 약한 위치 1곳 식별
   - 6/15까지 베이스라인 보고 `analytics/adsense-baseline-2026-06-15.md`

### P1 (Week 2로 이월 OK)

6. **GA Data API 셋업** (analytics)
   - Google Cloud 서비스 계정 → JSON 키 발급 → GA4 속성 권한 부여
   - `analytics/ga_client.ts` 스크립트로 일일 자동 추출 가능 상태

7. **신규 그룹 페이지 1개 추가** (seo + tribe-intel)
   - 4·5세대 핫 그룹 1개 선정 (TikTok 트렌드 기반)
   - JSON 추가 → Astro 자동 빌드 → Search Console 인덱싱 요청

### P2 (Week 1에 못해도 OK)
- Premium 광고 네트워크 (Mediavine, AdThrive) 조건 확인
- 다국어 hreflang 점검

### 팀장 발행 승인 프로토콜
- product가 신규 기능 1주 1개 → 팀장 리마커블 검토 → 통과해야 라이브
- adops가 광고 위치 변경 → analytics가 LCP/재방문율 영향 데이터 제출 → 팀장 승인 후 적용
