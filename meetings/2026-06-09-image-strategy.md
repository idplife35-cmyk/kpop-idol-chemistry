---

## 현황 진단 (analytics + seo 합동 보고)

코드 점검 결과:
- `GroupBanner.astro` — `logo` props가 있으면 `<img>` 표시, 없으면 그룹 이니셜 fallback ("BTS", "LE", "ZB1")
- `MemberGrid.astro` — `member.image` 있으면 `<img>`, 없으면 이모지 fallback (🐰, 💜, 🦋 등)
- `groups/*.json`의 `logo`: `/images/{group}_logo.png` 참조. 실제 `public/images/` 디렉토리 미확인 → **상당수 로고가 404**
- `members[].image`: 모든 그룹 JSON에서 미정의 → **현재 사이트는 100% 이모지로 멤버 표시**

즉, "이미지가 없다"는 사용자 진단은 정확. 현재 모든 멤버 카드가 이모지로 채워진 상태.

## 의제 1: 어떤 이미지를 만들 것인가? — 4가지 선택지

### 선택지 A — 실제 아이돌 사진 직접 게재
- 출처: 공식 SNS, 위키, 팬 커뮤니티
- 위험: K-Pop 회사(HYBE, SM, YG 등)의 IP. fan-made 면책 조항이 있어도 분쟁 가능. 이미 DMCA 사례 다수.

### 선택지 B — ChatGPT(DALL·E)로 실제 인물 닮은 이미지 생성
- 위험: 모델 거부 가능성 (실존 인물 likeness). 윤리적 회색지대. fan art 톤이면 통과해도 일관성 불가.

### 선택지 C — 추상 아바타 (그룹 컬러 + 멤버 이니셜 + 이모지)
- 안전. 코드로도 즉시 생성 가능 (SVG/Canvas).
- 단점: 진정성 약함. 부족이 "내 최애"를 못 알아봄.

### 선택지 D — 그룹 단위 컨셉 일러스트 (사람 없음, 세계관·미학 표현)
- DALL·E 안전 통과. EYEKONS·GLLIT·FEARNOT 미학을 추상화.
- og:image, 그룹 페이지 hero, ShareCard 배경 등에 재사용 가능.

## 각 에이전트 입장

**tribe-intel — D + C 조합 강력 지지**
- 부족 진정성 관점: 영어권 K-Pop 팬은 "내 최애" 사진을 보고 싶어함. 그러나 "내 최애의 가짜 닮은 이미지"는 부족 진정성 완전 손상 (B는 즉시 탈락).
- 대안: 그룹 미학(BLACKPINK = 핑크+블랙, NewJeans = 파스텔, KATSEYE = 글로벌) 컨셉 일러스트는 부족이 인지하는 시각 코드. EYEKONS가 KATSEYE 컬러+컨셉 보면 "내 그룹이다" 인지.
- 멤버 단위는 추상 아바타 (그룹 컬러 + 멤버 이니셜 + 멤버 고유 이모지). 사용자가 이미 이모지를 정체성 토큰으로 받아들이는 점 활용.

**product — D + C 채택, 단 ShareCard 우선**
- 현재 ShareCard는 4스타일 모두 이모지 + 그라데이션. 그룹 미학을 더 강하게 가져가면 공유성 ↑.
- 멤버 페이지 이미지보다 **결과 공유 카드 배경**이 임팩트 더 큼 (공유 화면이 친구에게 전달되는 자산이므로).
- 멤버별 이미지는 우선순위 낮음 — 멤버 페이지는 SEO 트래픽이지만 사용자는 결국 메인 폼으로 이동해서 케미 결과를 본다.

**ladder — C 보조 + D 메인**
- 즐겨찾기 패널에 멤버 이미지가 있으면 시각적 재방문 유도. 단 추상 아바타로 충분 (오히려 깔끔).
- PWA 홈 화면 아이콘 = 그룹 미학과 일치하는 D 자산이 핵심.

**seo — D 우선**
- og:image는 이미 200페이지 자동 생성됨 (빌드 산물). 페이지 hero 이미지는 "above the fold"라 LCP·CLS 영향.
- 그룹 페이지의 hero는 컨셉 일러스트 (1200×630 정도). 멤버 페이지의 hero는 그룹 컨셉 + 멤버 이니셜 오버레이 정도가 LCP 안전.
- 멤버 단위 개별 이미지 = LCP·총 빌드 크기 위협. 100여 페이지 × 200KB = 20MB 추가. **반대**.

**adops — D + C 그러나 광고 위 콘텐츠 강화 관점**
- 광고 단가는 페이지 시각적 풍부함에 영향. 빈약한 페이지는 RPM 하락.
- 그룹 hero에 컨셉 일러스트 1장 + 멤버 카드는 추상 아바타로 통일 = 광고 위 콘텐츠 충분.
- 단, **이미지 추가가 LCP를 3초 넘기면 즉시 롤백** (TEAM.md 금기 항목).

**analytics — A/B 테스트 가능성 강조**
- 현재 이모지 상태가 베이스라인. 일러스트 도입 후 7일 재방문율·공유율 변화 측정 가능.
- 단계적 도입 권고: 그룹 hero(D) 먼저 → 측정 → 멤버 카드(C) 결정.

## 비판적 검토 (Seth Godin 직접 점검)

**5가지 잘못될 경로**:

1. **"그룹 일러스트가 매끈하지 않아 진정성 손상"**
   → 대응: ChatGPT 프롬프트에 "fan zine illustration, slightly imperfect, hand-drawn feel" 톤. 광택보다 정성.

2. **"이미지 추가로 LCP 폭증"**
   → 대응: 모든 신규 이미지 1200×630 WebP < 80KB. Astro `<Image>` 컴포넌트로 자동 최적화 강제.

3. **"실인물 사진 유혹"**
   → 대응: TEAM.md 신규 금기 항목으로 명문화. PR 리뷰 시 즉시 거부.

4. **"멤버별 이미지 양산 → 페이지 부풀림"**
   → 대응: Phase 1은 그룹 단위(22개)만. 멤버 단위는 7일 데이터 검증 후.

5. **"DALL·E 거부로 큐 정체"**
   → 대응: 프롬프트는 "no real person, abstract/symbolic" 명시. 실패한 큐는 자동으로 done/에서 fail/로 분리.

## 팀장 결정 (Seth Godin)

> **결정 1**: 선택지 D (그룹 컨셉 일러스트) + 선택지 C (멤버 추상 아바타) 조합.
> **결정 2**: Phase 1은 그룹 hero 일러스트 22개만. 멤버 카드는 추상 아바타로 일괄 자동 생성 (코드).
> **결정 3**: 선택지 A·B는 **TEAM.md 신규 금기 항목**으로 명문화.

**근거 (Seth Godin 체크리스트)**:
1. 부족 봉사: 컨셉 일러스트 = 그룹 미학 인지, 추상 아바타 = 멤버 식별 토큰. 둘 다 통과.
2. 리마커블: "AI가 만든 가짜 닮은 사진"이 아니라 "팬진 일러스트" 톤이면 공유 시 진정성 유지.
3. 농부 마인드셋: 100개 페이지 이미지 양산 금지. 22개로 시작, 데이터 보고 확장.
4. 안전: 저작권·AI 윤리 회색지대 진입 금지.

## 실행 계획

### Phase 1 (Week 24, 즉시)
1. **`tools/image_gen/queue/`에 22개 그룹 컨셉 일러스트 큐 자동 생성**
   - 스크립트: `tools/image_gen/seed_group_queue.mjs` (자동 큐 시드)
   - 각 그룹 JSON의 color·description·fandom을 프롬프트에 주입
   - 출력 경로: `public/images/groups/{slug}-hero.webp` (1200×630, < 80KB 목표)

2. **멤버 추상 아바타 자동 생성 (코드)**
   - 컴포넌트: `MemberAvatar.astro` — SVG로 즉시 렌더 (이미지 파일 미생성)
   - 패턴: 그룹 컬러 그라데이션 배경 + 멤버 이니셜(흰색) + 멤버 이모지(우측 하단)
   - LCP 영향 zero (SVG 인라인)

3. **`MemberGrid.astro`를 MemberAvatar로 갱신**
   - 기존 이모지 fallback을 MemberAvatar로 교체
   - `member.image`가 있으면 그대로 사용 (미래 확장 여지)

4. **groups/*.json의 `logo` 필드 갱신**
   - 모든 그룹의 `logo`를 `/images/groups/{slug}-hero.webp`로 가리킴
   - 빌드 시 누락 이미지는 ininitials fallback (기존 동작 유지)

### Phase 2 (Week 25 데이터 검증 후)
5. analytics가 A/B로 측정: hero 일러스트 도입 페이지의 7일 재방문율·공유율
6. 양호 시 → 멤버 단위 일러스트 검토 (1~2 그룹 시범)
7. 부족 진정성 손상 신호 시 → 즉시 롤백
