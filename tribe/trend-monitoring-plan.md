# 트렌드 모니터링 · 콘텐츠 발행 계획 v1

> 작성: 부족 인텔리전스 (tribe-intel)
> 1차 부족: 영어권 K-Pop 팬 (4·5세대 70% / 1세대 30%) — `worldview-v1.md`
> 발행 게이트키퍼: Seth Godin (팀장 리마커블 검토)
> 갱신: 매주 일요일

---

## 운영 원칙

1. **트렌드는 신호이지 발행 명령이 아니다.** 모든 트렌드는 "이 트렌드가 우리 부족(영어권 K-Pop 팬)의 정체성 토큰이 되는가?"를 통과해야 발행.
2. **모니터링 → 해석 → 발행 후보 → 게이트 → 발행** 5단계. 각 단계에 명시적 게이트.
3. **자동 수집 + 수동 큐레이션.** API/스크래핑으로 raw 신호를 모으고, tribe-intel이 매주 한 번 의미를 해석.
4. **수량 금지.** 행궁동 패턴: 5개 후보 → 팀장 검토 → 통과 2~3개만 발행. AI 양산 절대 금지.

## A. 모니터링 소스 (4축)

### 축 1. SNS 트렌드 — "지금 무엇이 공유되는가"

| 소스 | 방식 | 빈도 | 신호 의미 |
|------|------|------|----------|
| **Twitter/X 트렌드** | Trends24, GetDayTrends 스크래핑 (KR/US/JP 지역) | 일 1회 | 컴백·이슈·MV 반응 |
| **TikTok 디스커버리** | Playwright + 키워드 검색 ("#kpop", "#bts", 그룹별) | 주 2회 | 댄스 챌린지·바이럴 영상·신규 곡 인기 |
| **Reddit r/kpop** | RSS (`reddit.com/r/kpop/top/.rss?t=week`) | 일 1회 | 영어권 팬덤 토론 주제 |
| **YouTube 트렌딩 — Music (KR)** | YouTube Data API v3 (videos.list, chart=mostPopular) | 일 1회 | MV 조회수·신규 곡 진입 |
| **Weverse 공지 모음** | 공식 RSS or 스크래핑 (그룹별) | 일 1회 | 컴백·이벤트 1차 발표 |
| **Instagram Stories** | (수동) tribe-intel이 직접 팔로우 그룹 7~10개 | 주 1회 회고 | 일상 톤·콘셉트 변화 |

**자동화 스크립트**: `analytics/trend_pull.mjs` (별도 작성 — 본 문서 § E)

### 축 2. 공식 일정 — "다음 4~6주에 무엇이 일어나는가"

| 소스 | 수집 항목 | 빈도 |
|------|----------|------|
| **Wikipedia "List of upcoming K-pop releases"** | 컴백 일자·앨범명·트랙리스트 | 주 1회 |
| **Allkpop "Comeback Calendar"** | 컴백·뮤직쇼 출연 | 주 1회 |
| **Weverse Concerts** | 영어권 투어 일정 (북미·유럽·동남아) | 주 1회 |
| **JYP / HYBE / SM / YG IR 자료** | 공식 분기 보고서의 활동 계획 | 분기 1회 |
| **K-pop Wiki Discography** | 멤버 솔로/유닛 활동 | 월 1회 |

→ 산출: `tribe/calendar-{YYYY-MM}.md` — 4·5세대 + 1세대 핵심 그룹 컴백·콘서트 캘린더, 4주 룩어헤드.

### 축 3. 콘서트·이벤트 — "영어권 트래픽 가속 트리거"

콘서트는 영어권 트래픽의 **순간 증폭 트리거**. 단순 일정이 아니라 **국가별·도시별 트래픽 예측**.

| 이벤트 유형 | 트래픽 예측 신호 | 우리 액션 |
|------------|----------------|----------|
| 미국 월드투어 (스타디움급) | US, CA 트래픽 +30~50% 가능 | 해당 그룹 페이지 SEO·공유 카드 우선 점검 |
| 일본·동남아 투어 | JP, PH, ID 트래픽 +20~40% | 일본어 페이지 추가 검토 트리거 |
| Coachella·Glastonbury 등 글로벌 페스티벌 | 글로벌 유입 단일 피크 | 페스티벌 전 1주 콘텐츠 발행 타이밍 |
| 팬미팅 (소규모) | 도시 단위 트래픽 +10% | 우선순위 낮음 |

### 축 4. 아이돌별 추구 방향 (SNS 콘텐츠 해석)

각 그룹의 "지금 추구하는 세계관"을 SNS 업로드 패턴으로 해석한다.

**해석 프레임 (그룹별 매주 갱신)**:
```
GROUP: NewJeans
이번 주 업로드 톤: [밝음 / 차분 / 콘셉트 강화 / 일상]
키워드 빈도: ["bunnies", "summer", "pre-debut", ...]
멤버별 노출: Minji 30%, Hanni 25%, Danielle 20%, ...
추구 정체성: "Y2K + summer escapism"
우리 사이트와 연결점:
  - 결과 카드 컬러 무드 동조 (라이트 블루·여름 톤)
  - 부족 큐레이션 콘텐츠: "Bunnies summer essentials"
```

**자동 추출 가능 vs 수동 필요**:
- 자동: 업로드 빈도, 멤버별 등장 비율, 캡션 키워드 빈도 (Instagram public scraping)
- 수동: "추구 정체성" 해석 (tribe-intel의 사람·AI 협업 판단)

## B. 콘텐츠 발행 결정 흐름 (5단계 게이트)

```
[1단계 — 수집]
   trend_pull.mjs가 4축에서 raw 신호 수집 → analytics/trends/YYYY-MM-DD.json

[2단계 — 후보 추출 (tribe-intel 주간)]
   매주 일요일: 다음 주 발행 후보 5개 추출.
   기준: ① 영어권 부족과 정합, ② 우리 사이트 도구와 연결 가능, ③ 4·5세대 우선

[3단계 — 게이트 (Seth Godin 체크리스트)]
   각 후보에 대해 자문:
   ☐ 누구의 어떤 문제를 해결하는가? "모든 K-Pop 팬"이면 탈락
   ☐ 카드/콘텐츠가 친구에게 공유될 이유가 있는가? "매우 좋은 것"이면 탈락
   ☐ 퍼널 안에 위치하는가? (검색 유입 → 우리 도구 → 공유)
   ☐ 부족 정체성 토큰인가? (자랑·소속감)
   통과 ≥ 3개 항목 → 발행. 그 미만 → 폐기 또는 재기획

[4단계 — 발행 형식 선택]
   유형 A: 신규 그룹 페이지 추가 (가장 SEO 강함)
   유형 B: 블로그 글 (롱테일 SEO + 깊이)
   유형 C: 신규 카드 스타일 (공유성 + 리마커블)
   유형 D: 결과 화면 카피 변형 (A/B 테스트)
   유형 E: 캐릭터/그룹별 한정 기능 (시즌 컴백용)

[5단계 — 측정 후 회고]
   발행 후 7일: analytics가 공유 클릭·신규 유입·재방문 영향 보고
   2개 연속 발행이 공유 0 → Seth Godin 금기 발동: 기준 재정의
```

## C. 주간 발행 리듬 (4주 사이클)

| 주차 | 주제 | 산출물 (목표 2~3건) |

## F. 사용자 정책 부합 (재확인)

사용자 지시: **"항상 통계 기반으로 의사결정"**, **"API가 유료라면 웹페이지로"**

- Reddit RSS: 무료 ✅
- YouTube Data API v3: 무료 할당 (10,000 unit/일) ✅
- Wikipedia: 무료 ✅
- Twitter/X API: 유료/제한 → **Playwright 스크래핑** (사용자 정책 부합)
- TikTok 공식 API: 제한적 → **Playwright 스크래핑**
- Instagram: 공식 API는 비즈니스 계정 제약 → **Playwright public 페이지 스크래핑**

→ 모든 모니터링은 무료 채널로 가능. Playwright는 `analytics/ga_scraper.mjs`와 동일 패턴 재사용.
